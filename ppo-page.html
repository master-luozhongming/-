    06 PPO：稳一点的策略更新¶

    ?? 一句话导读：上一章的策略梯度法有个要命的毛病——步子一迈大就学崩，崩了还要花很久才能爬回来。这一章讲的近端策略优化（PPO，Proximal Policy Optimization，说人话就是"每次只准小步往前挪，别一脚踩空"），用一个叫裁剪（clip）的小机关把更新幅度锁死在安全范围内。学完你会明白：为什么现在训练 ChatGPT 这类模型做 RLHF，几乎都默认用 PPO——因为它又稳又好调。

    ?? 先讲个直觉¶

    想象你在教一个刚学会站立的小孩走路。

    你的目标是让他走得又快又稳。于是你扶着他，让他往前迈步：

    • 迈得太小：磨磨蹭蹭，半天挪不了一米，学得太慢。
    • 迈得太大：重心一下子全压过去，"啪"地摔个大跟头——摔了之后他比没学之前还怂，得重新哄半天才肯再站起来。

    强化学习里的"策略"就是这个学步的小孩。策略梯度法（上一章）就像一个不会看路况的大人，每一步都按固定力气往前推。 路平的时候没事，可一旦前面是个台阶或者陡坡，同样的力气就会把小孩直接推下去。

    而走钢丝的杂技演员给了我们另一种思路：他从不大跨步。每一步都先用脚尖探一探，确认脚下这一小块区域是安全的，才把重心挪过去——幅度被牢牢限制在一个小圈里。

    PPO 干的就是这件事：不管前面地形多险，每次更新都只准在"上一版策略附近的一个小圈圈里"挪动。 既往前进步了，又绝不会一步迈太大把自己作死。这个"小圈圈"，就是它名字里"近端（Proximal，意思是'离原地不远'）"的来历。

    ?? 核心概念（说人话）¶

    为什么策略梯度法这么容易崩？¶

    先回顾上一章：策略梯度法是"试一把 → 看结果好坏 → 把好动作的概率调高、坏动作调低"，然后用梯度上升朝着"奖励更高"的方向推一步。

    这里藏着一个深度学习里没有、但强化学习里特别毒的问题：

    深度学习的训练数据是死的，强化学习的训练数据是活的。

    • 训练一个图像分类器，不管你的参数怎么变，那一万张猫狗照片永远摆在那不动。参数和数据互不影响。
    • 但强化学习里，你的训练数据（轨迹）是你自己当前的策略"玩"出来的。策略一变，下一轮采集到的数据也跟着变。

    这就形成一个恶性循环：步子迈大了 → 策略变差 → 用这个差策略去玩，采集到一堆烂数据 → 拿烂数据继续训练 → 越训越差。一步踏空，满盘皆输，而且爬不回来。这就是为什么 RL 训练曲线经常"哗"地一下崩到底。

    优势（Advantage）：这个动作比平均水平好多少¶

    PPO 是 Actor-Critic 架构（演员-评论家架构，说人话就是"一个演员负责做动作，一个评论家负责打分"）。评论家给出的关键数字叫优势（Advantage，记作 \(A_t\)）。

    人话定义：优势 = "在这个局面下，我选的这个动作，比'随便平均一下能拿到的分'高出多少"。

    • \(A_t > 0\)：这步走得比平均水平好 → 应该鼓励，以后多这么干。
    • \(A_t < 0\)：这步比平均还差 → 应该打压，以后少这么干。

    记住这一点，下面的裁剪规则全靠它。

    比值 ρ：新策略和旧策略的"差异表"¶

    PPO 每轮会先用旧策略玩一局，把轨迹存下来。然后训练新策略时，针对同一个局面 \(s_t\)、同一个动作 \(a_t\)，比一比新旧两个策略给这个动作的概率：

    比值（ratio，记作 \(\rho_t\)）= 新策略给这动作的概率 ÷ 旧策略给这动作的概率。

    • \(\rho_t = 1\)：新旧策略一模一样，没变化。
    • \(\rho_t = 1.5\)：新策略比旧策略更爱这个动作（概率涨了 50%）。
    • \(\rho_t = 0.5\)：新策略没那么爱这个动作了（概率砍了一半）。

    PPO 的整套机制，就是盯着这个比值，不准它离 1 太远。

    裁剪（Clip）：给步子装个"限位器"¶

    裁剪就是个简单粗暴的"限位器"：给比值 \(\rho_t\) 设一个上下限 \([1-\varepsilon,\ 1+\varepsilon]\)（论文里 \(\varepsilon=0.2\)，也就是只准在 0.8 到 1.2 之间晃）。

    • 比值想冲到 1.5？对不起，按 1.2 算。
    • 比值想跌到 0.3？对不起，按 0.8 算。

    一旦比值被"摁"在边界上，它对应的那块目标就变成了一个常数——常数的梯度是 0，于是模型在这个方向上"再也使不上劲"，自然就不会继续往外冲。这就是 PPO 防止"迈大步"的核心机关。

    ?? 必要的公式¶

    先看上一章策略梯度法的目标（Actor-Critic 版）长这样：

    \[ \nabla_\theta J(\theta) = \mathbb{E}\left[\sum_t A_t \,\nabla_\theta \log \pi_\theta(a_t \mid s_t)\right] \]

    ?? 人话翻译：朝着"让优势 \(A_t\) 大的动作、概率涨得更多"的方向推。问题就在于——它对"推多远"完全没设防。

    PPO 把目标换成了带裁剪的替代目标函数（surrogate objective）：

    \[ J^{\text{clip}}(\theta) = \mathbb{E}\left[\sum_t \min\Big(\rho_t\, A_t,\ \ \text{clip}(\rho_t,\, 1-\varepsilon,\, 1+\varepsilon)\, A_t\Big)\right] \]

    其中比值 \(\rho_t = \dfrac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{\text{old}}}(a_t \mid s_t)}\)。

    ?? 人话翻译：括号里有两项——

    • 左边 \(\rho_t A_t\)：原汁原味的"按比值放大优势"，不设防。
    • 右边 \(\text{clip}(\dots)A_t\)：把比值先限位到 [0.8, 1.2] 再乘优势。

    最后取两者里更小（min）的那个当目标。这个 min 是精髓：它保证模型永远拿不到"迈大步"的好处——一旦你想靠"猛增某动作概率"来多捞分，min 会把你的收益按裁剪后的天花板掐住，多冲也是白冲。等于告诉模型："小步走有奖，大步走没奖，那你还冲个啥。"

    一个漂亮的彩蛋：第一步就是普通策略梯度

    训练刚开始时新策略 = 旧策略，所以比值 \(\rho_t = 1\)，落在 [0.8, 1.2] 之内，完全不触发裁剪。这时对它求导，会发现 PPO 的第一次更新和上一章那个朴素的策略梯度法一模一样。PPO 不是推翻了策略梯度，而是给它加了个"安全带"。

    想看 min + clip 的 6 种情况分类讨论（可跳过）

    设比值为 \(\rho_t\)、优势为 \(A_t\)，逐一看什么时候有梯度、什么时候梯度被掐成 0：

    情况 比值 \(\rho_t\) 优势 \(A_t\) 取到的值 被裁剪？ 有梯度？
    1 在 \([1-\varepsilon, 1+\varepsilon]\) 内 + \(\rho_t A_t\) 否 ? 正常更新
    2 在 \([1-\varepsilon, 1+\varepsilon]\) 内 ? \(\rho_t A_t\) 否 ? 正常更新
    3 \(< 1-\varepsilon\) + \(\rho_t A_t\) 否 ? 朝范围内拉，更新
    4 \(< 1-\varepsilon\) ? \((1-\varepsilon)A_t\) 是 ? 梯度=0
    5 \(> 1+\varepsilon\) + \((1+\varepsilon)A_t\) 是 ? 梯度=0
    6 \(> 1+\varepsilon\) ? \(\rho_t A_t\) 否 ? 朝范围内拉，更新

    一句话归纳：只在两种时候更新策略——(a) 比值还在安全圈 [0.8, 1.2] 内；(b) 比值已经跑出圈了，但优势的方向正好是"把它往圈里拉回来"。

    最反直觉的是情况 5：比值已经 > 1.2（新策略已经比旧策略大大增加了这个动作），优势又是正的——按朴素策略梯度它还想继续加码。但 PPO 说："你已经够贪了，别再冲了"，于是梯度=0，不更新。这正是"防止迈大步"的体现。

    （为什么裁剪后梯度=0？因为 \((1\pm\varepsilon)\) 和 \(A_t\) 都是不含 \(\theta\) 的常数，常数对 \(\theta\) 求导当然是 0。）

    ?? 代码长什么样¶

    PPO 每一轮的核心就两件事：用旧策略采一批数据，然后拿这批数据把新策略小步更新 N 次。

    # 一轮 PPO 的主干（伪代码）
epsilon = 0.2                              # 裁剪阈值，舒适区 = [0.8, 1.2]

traj = run_policy(pi_old)                  # 1) 用旧策略玩一局，存下轨迹
advantages = estimate_advantage(traj)      # 2) 评论家算每步的优势 A_t（常用 GAE）

pi_new = clone(pi_old)                     # 3) 新策略从旧策略出发
for i in range(N):                         #    拿同一批数据反复更新 N 次
    for (s, a, A) in zip(traj.states, traj.actions, advantages):
        # 新旧策略对"同一个动作"给的概率之比
        ratio = pi_new.prob(a, s) / pi_old.prob(a, s)
        # 限位器：把比值摁进 [0.8, 1.2]
        clipped = clip(ratio, 1 - epsilon, 1 + epsilon)
        # 取较小者——拿不到"迈大步"的好处
        obj = min(ratio * A, clipped * A)
    obj.backward()                         # 反向传播求梯度
    optimizer.step()                       # 朝"目标更大"的方向小步挪

pi_old = pi_new                            # 4) 新策略变成下一轮的旧策略

    几个会反复出现的小词

    • GAE（Generalized Advantage Estimation，广义优势估计）：算优势 \(A_t\) 的标准方法，目的是让优势这个数字"既准又不抖"。
    • N 步更新：旧策略辛辛苦苦采的一批数据，PPO 舍不得用一次就扔，会重复利用 N 次——这也是它比朴素策略梯度省数据的原因之一。

    ??? 一图看懂¶

    flowchart TD
    A["旧策略 π_old<br/>玩一局，存下轨迹"] --> B["评论家算优势 A_t<br/>(这步比平均好多少)"]
    B --> C["新策略 π_new 上场<br/>算比值 ρ = 新概率 / 旧概率"]
    C --> D{"ρ 在舒适区<br/>[0.8, 1.2] 内吗?"}
    D -->|"在区内"| E["正常更新<br/>(放心往前挪小步)"]
    D -->|"冲出区外<br/>且越冲越远"| F["裁剪 → 梯度=0<br/>(限位器卡住，原地不动)"]
    D -->|"在区外<br/>但优势把它往回拉"| E
    E --> G["小步更新 N 次后<br/>新策略 → 变成下一轮旧策略"]
    F --> G
    G -.下一轮.-> A

    把它和走钢丝对照：比值 \(\rho\) 就是"你这一步迈了多远"，舒适区 [0.8, 1.2] 就是"脚尖能安全够到的范围"。一旦你想迈出这个范围，限位器（裁剪）立刻把你的"使劲"清零——你最多只能挪到边界，绝不会一脚踩空。

    ?? 产品经理 take¶

    • PPO 是当今 RLHF 的"默认引擎"。ChatGPT、Claude 这类模型做"用人类偏好对齐"（RLHF）时，背后调参数的主力算法多年来就是 PPO。理由不是它效果最炸，而是它最省心——稳定、好调、不容易莫名其妙崩盘。工程上，"不会半夜训崩"比"理论上更优"值钱得多。
    • 它本质是在"学得快"和"别学崩"之间做权衡。\(\varepsilon\)（那个 0.2）就是你能拧的旋钮：调小 → 更稳但更慢（步子更碎）；调大 → 更快但更容易翻车。这和上一章学习率"太大崩、太小慢"的两难是同一回事，只不过 PPO 把这个两难关进了笼子，让你不至于一调就炸。
    • "近端"思想到处都能借用。任何"基于反馈反复迭代"的系统——A/B 测试调产品、推荐策略灰度上线、运营规则迭代——都该警惕"一次改太猛"。PPO 给的产品启示就是：每次只在上一版附近小步快跑，比偶尔一次激进调整更不容易翻车。灰度发布、限制单次改动幅度，本质都是工程版的"裁剪"。
    • 坑在哪：PPO 不保证每一步都严格变好（它只是个"软约束"，偶尔还是会做错决策），而且它要同时训练演员和评论家两个网络，调起来零件比策略梯度多。但综合"实现简单 + 效果稳 + 数据可复用"，它依然是性价比最高的选择。

    一句话记住

    PPO = 给策略更新装了个"限位器"：每次只准在上一版策略的小圈圈里挪一步，想迈大步？梯度直接清零——既往前进步，又绝不踩空摔崩。
