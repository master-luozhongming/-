## 简单介绍

强化学习已成为增强大语言模型初始训练效果的强大工具，尤其是在推理密集型任务中。DeepSeek 最近在 DeepSeek-Math 和 DeepSeek-R1 模型上取得的突破，展现了 RL 在提升 LLM 数学推理和问题解决能力方面的巨大潜力。

这些成就得益于一种名为“组相对策略优化”（GRPO）的创新强化学习方法，该方法解决了将强化学习应用于语言模型的独特挑战。在本文中，我们将深入探讨 GRPO 的工作原理，以及它为何代表了 LLM 训练的重大进步。

## PPO vs GRPO

![[ppovsgrpo.excalidraw|1000]]


近端策略优化（PPO）一直是强化学习微调大语言模型的首选算法。PPO 的核心是一种策略梯度方法，它使用裁剪来限制策略更新（梯度），从而防止策略发生破坏性的大幅变化。PPO 的目标函数可以写成：

![[PPO目标函数-手写风格.png|PPO算法的目标函数]]

GRPO 建立在 PPO 的基础上，但引入了几项关键创新，使其更高效 、更适合微调大语言模型：

1. 无需价值函数网络，因此 **减少内存/计算** 使用量
2. 通过采样一组输出进行更有效的稳定 **优势估计**
3. 通过进一步惩罚目标和奖励，使用更保守的更新机制

## 进一步学习GRPO

![[grpo-in-detail.excalidraw|1000]]

### LLM作为策略

在 GRPO 中，大语言模型充当策略网络（actor，演员），将问题 $q$ 作为输入观察 $s$ ，并生成一系列 token 作为动作（action）。策略分布涵盖了各个 token：

$$
\pi_{\theta}(a|q)=\prod_{t=1}^N \pi_{\theta}(a_{t}|q,a_{<t})
$$

```ad-note
在原论文中，他们使用 $o_t$ 来表示时间步 $t$ 的输出 token 。而我们使用 $a_t$ 来符合 RL 中 action 的标准符号。
```

## 顺序生成token

由于 Transformer/LLM 的自回归特性，生成过程本质上是连续的：

1. 每个 token 都是根据之前的 token 有条件生成的
2. 策略网络（LLM）维护运行上下文
3. 每个 token 生成步骤都可以看作是 RL 框架中的一个动作 $a_t$ 。

![[grpo-in-detail-1.excalidraw|1000]]
## 奖励和优势计算

对于每个生成的序列，GRPO 按如下方式计算每个token的奖励：

$$
r_{t}=r_{\phi}(s,a_{\leq t})-\beta\log \frac{\pi_{\theta}(a_{t}|s,a_{<t})}{\pi_\text{ref}(a_{t}|s,a_{<t})}
$$

GRPO不使用价值函数网络，而是使用同一个问题作为输入，然后使用**参考模型**输出一组回答（多个回答），每个回答都有奖励，对一组回答的奖励进行归一化来估计基线优势 $A$：

$$
\hat{A}_{i,t}=\tilde{r}_{i}=\frac{r_{i}-\text{mean}(r)}{\text{std(r)}}
$$

## GRPO的目标函数

对于每个问题 $q$，GRPO从旧策略 $\pi_{\theta_\text{old}}$ 中采样一组输出 $o_1,o_2,\dots,o_G$，然后通过最大化 GRPO 目标函数来优化策略模型。完整的 GRPO 目标函数将所有内容整合在一起：

![[GRPO目标函数-手绘风格.png]]

目标函数包括：

1. 一组回答的各种平均值（回答的序列长度的均值，奖励的均值等等）
2. 使用裁剪进行保守更新
3. 包括对KL散度的**估计**，作为惩罚，以防止与参考模型出现较大偏差

## 结论

GRPO 代表了强化学习在大语言模型中的应用取得了重大进展。它消除了对价值函数网络的需求，并引入了组相对优势估计，从而提供了更高效、更稳定的训练流程。DeepSeek-Math 和 DeepSeek-R1 的成功证明了这种方法的实际效益。

GRPO 的关键创新——组采样、相对优势估计以及价值函数网络的消除——为 LLM 训练的未来发展提供了蓝图。随着我们不断突破语言模型的极限，像 GRPO 这样的技术对于充分释放其潜力至关重要。
