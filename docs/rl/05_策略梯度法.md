## 1. 一句话

**直接调「做决策的网络」，让高分动作概率变大、低分动作概率变小。**  
整章只改一个东西：权重 Φ_t 怎么算。

## 2. 母公式（全章骨架）

\[
\nabla_\theta J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}\left[\sum_{t=0}^{T} \Phi_t \,\nabla_\theta \log \pi_\theta(A_t \mid S_t)\right]
\]

- `∇ log π`：让当前动作更可能被选中
- `Φ_t`：这步动作好不好——**五代算法唯一区别**

## 3. 五代进化（对照表）

| 版本 | Φ_t | 解决什么问题 |
|------|-----|--------------|
| ① 原始 PG | 整局总分 G(τ) | 起点；异常高分会导致更新爆炸 |
| ② REINFORCE | 从 t 往后的分 G_t | 去掉「过去奖励」噪声 |
| ③ **带基线** | G_t − b(S_t) | 全正奖励时也能区分好坏 |
| ④ Actor-Critic | δ = R_t + γV(S_{t+1}) − V(S_t) | 不等整局结束，走一步学一步 |
| ⑤ GAE | 多步 TD 误差的 λ 加权 | 偏差与方差之间的旋钮 |

---

## 4. 带基线（Baseline）— 重点

### 4.1 问题从哪来

CartPole 每步奖励都是 **+1** → REINFORCE 的 G_t 永远为正 → 差动作也会被抬高概率。

### 4.2 做法

\(\Phi_t = G_t - b(S_t)\)，常用 \(b(S_t) = V(S_t)\)。

### 4.3 实现

```python
# REINFORCE：loss += -log_prob * G
# 带基线：  loss += -log_prob * (G - V(s).detach())
```

减基线无偏（期望不变）但降方差。详见原书定理 5.3。
